# README Branding Polish Design

## Goal
Improve Rocket branding in `README.md` while keeping onboarding concise and practical.

## Approved Direction
Use a premium hero block:
- centered Rocket logo
- strong product title and short tagline
- compact capability badges
- preserve quick-start-first structure

## Scope
In scope:
- Visual and content polish of README header and intro
- Keep existing quick start, docs links, and command sections

Out of scope:
- New marketing pages/assets
- Major restructure of README sections

## UX Principles
- First screen should communicate identity and value quickly.
- Avoid long prose before setup commands.
- Keep repository users focused on "run backend + run frontend" path.

## Validation
- Markdown renders cleanly on GitHub.
- Existing local path references remain valid.
- Quick Start remains near top of file.
